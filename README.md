# 王大同的個人作品集

這是我建立的互動式網頁設計實驗作品集，包含了不同階段的 Web Labs。

## 🚀 如何將此專案上傳至 GitHub 並轉換為個人網站 (使用 GitHub Desktop)

請依照以下步驟操作，將這份本地網頁作品集轉換成網路上大家都能看到的個人網站：

### 第一階段：透過 GitHub Desktop 上傳程式碼至 GitHub
1. **開啟 GitHub Desktop** 軟體，如果你還未登入，請先使用你的 GitHub 帳號登入。
2. 點擊軟體左上角的 **File** -> **Add Local Repository...** (新增本地數據庫)。
3. 在彈出視窗的 `Local path` 欄位中，點擊 **Choose...** 並選擇你桌面上的 `imweb` 資料夾。
4. 這時軟體可能會提示「This directory does not appear to be a Git repository.」，請直接點擊下方藍色字體的 **「create a repository」**。
5. 確認專案名稱為 `imweb` 後，點擊 **Create repository**。
6. (如果有的話) 在軟體左下角的 **Summary** 欄位輸入提交名稱（例如：「`第一版網頁上傳`」），然後點擊 **Commit to main** (或 master)。
7. 最後，點擊軟體頂部的 **Publish repository** 按鈕。
   - ⚠️ **重要提醒**：在發佈前彈出的視窗中，請記得 **取消勾選** 「Keep this code private」(保持私有) 的選項，這樣稍後你的網頁才能公開被大眾瀏覽。然後再按下 Publish。

### 第二階段：設定 GitHub Pages (開啟網頁功能)
1. 打開瀏覽器，前往 [GitHub 網站](https://github.com/) 登入你的帳號，並進入你剛剛上傳的 `imweb` 專案頁面。
2. 在專案頁面上方選單，點選最後一個 **Settings** (齒輪圖示設定)。
3. 在左側長長的選單中，往下拉找到 **Pages** 選項並點選。
4. 在中間的 **Build and deployment** 區塊：
   - 找到 **Branch** (分支) 設定，將下拉選單的 `None` 改成 `main` (或是 `master`)。
   - 點擊旁邊的 **Save** 按鈕。
5. **等待與完成**：設定完成後，GitHub 會花大概 1 到 3 分鐘來建置你的網頁。稍候重新整理 Pages 頁面，最上方就會出現一段成功提示：
   > 🎉 **"Your site is live at `https://你的帳號.github.io/imweb/`"**
6. 點擊該網址，恭喜！你的個人網站就正式上線了！
